# Quick Start Guide\n\nGet MileyMusics running locally in 5 minutes.\n\n## Prerequisites\n\n- **Node.js** >= 18.0.0 (download from nodejs.org)\n- **npm** or **yarn** (comes with Node.js)\n- **Git** (download from git-scm.com)\n- **Code editor** (VS Code recommended)\n\n## 1️⃣ Clone & Install\n\n```bash\n# Clone repository\ngit clone https://github.com/Raghav549/mileymusics.git\ncd mileymusics\n\n# Install frontend dependencies\nnpm install\n\n# Install backend dependencies\ncd backend\nnpm install\ncd ..\n```\n\n## 2️⃣ Setup Supabase\n\n1. Create free account at [supabase.com](https://supabase.com)\n2. Create new project\n3. Wait for database provisioning (~2 minutes)\n4. Go to **Settings → API** and copy:\n   - `Project URL` → `VITE_SUPABASE_URL`\n   - `anon public key` → `VITE_SUPABASE_ANON_KEY`\n\n## 3️⃣ Configure Environment\n\n### Frontend\n\n```bash\n# Create .env.local\necho 'VITE_SUPABASE_URL=YOUR_URL\nVITE_SUPABASE_ANON_KEY=YOUR_KEY\nVITE_BACKEND_URL=http://localhost:5000' > .env.local\n```\n\n### Backend\n\n```bash\n# Create .env\ncd backend\ncp .env.example .env\n\n# Edit .env and add:\n# SUPABASE_URL=YOUR_URL\n# SUPABASE_ANON_KEY=YOUR_KEY\n# SUPABASE_SERVICE_KEY=YOUR_SERVICE_KEY (from Settings → API → Service Role Key)\n```\n\n## 4️⃣ Setup Database\n\n```bash\n# Run migrations in Supabase SQL Editor\n# Go to Supabase dashboard → SQL Editor\n# Create new query and paste contents of:\n# - backend/migrations/001_init_schema.sql\n# - backend/migrations/002_rls_policies.sql\n# - backend/migrations/003_triggers.sql\n```\n\nOr via command line:\n```bash\ncd backend\npsql -h your-project.supabase.co -U postgres < migrations/001_init_schema.sql\npsql -h your-project.supabase.co -U postgres < migrations/002_rls_policies.sql\npsql -h your-project.supabase.co -U postgres < migrations/003_triggers.sql\n```\n\n## 5️⃣ Start Development Servers\n\n### Terminal 1 - Frontend\n```bash\nnpm run dev\n# Opens http://localhost:5173\n```\n\n### Terminal 2 - Backend\n```bash\ncd backend\nnpm run dev\n# Runs on http://localhost:5000\n```\n\n## ✅ Verify Installation\n\n1. **Frontend**: http://localhost:5173\n   - Should see login page\n   - Register new account\n\n2. **Backend Health**:\n   ```bash\n   curl http://localhost:5000/health\n   # Should return: {\"status\":\"ok\"}\n   ```\n\n3. **Database Connection**:\n   ```bash\n   curl http://localhost:5000/api/health/db\n   # Should return: {\"status\":\"connected\"}\n   ```\n\n## 🚀 Try It Out\n\n1. **Create Account**\n   - Go to http://localhost:5173\n   - Click \"Sign Up\"\n   - Enter email, password, username\n   - Click \"Create Account\"\n\n2. **Upload Music**\n   - Click \"Upload\"\n   - Select MP3 file\n   - Add title, description, genre\n   - Click \"Publish\"\n\n3. **Explore**\n   - Click \"Browse\" to see all songs\n   - Click \"Search\" to find music\n   - Click heart to like songs\n\n## 📚 Useful Commands\n\n```bash\n# Frontend\nnpm run dev           # Start dev server\nnpm run build        # Build for production\nnpm run lint         # Check code style\n\n# Backend\ncd backend\nnpm run dev          # Start dev server\nnpm run seed         # Populate with sample data\nnpm test             # Run tests\nnpm run lint         # Check code style\n```\n\n## 🐛 Troubleshooting\n\n### Port already in use\n```bash\n# Kill process using port 5000\nsudo lsof -ti:5000 | xargs kill -9\n\n# Or change port in backend/.env\nPORT=5001\n```\n\n### Supabase connection error\n- Check `.env` or `.env.local` has correct URLs\n- Verify keys are copied completely\n- Check internet connection\n- Test: `curl https://your-project.supabase.co/rest/v1/`\n\n### Database migration error\n- Make sure you're using the right PostgreSQL user (postgres)\n- Check all three migrations ran without errors\n- Verify Supabase project was created successfully\n\n### Frontend blank page\n- Open browser DevTools (F12)\n- Check Console for errors\n- Verify Supabase credentials\n- Check backend is running\n\n### npm install fails\n- Clear cache: `npm cache clean --force`\n- Delete node_modules: `rm -rf node_modules`\n- Delete lock file: `rm package-lock.json`\n- Try again: `npm install`\n\n## 📖 Next Steps\n\n1.
