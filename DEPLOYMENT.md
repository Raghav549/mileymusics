# Deployment Guide\n\nComplete deployment instructions for MileyMusics frontend and backend.\n\n## Table of Contents\n1. [Frontend Deployment (Vercel)](#frontend-deployment-vercel)\n2. [Backend Deployment (Railway)](#backend-deployment-railway)\n3. [Database Setup](#database-setup)\n4. [Environment Configuration](#environment-configuration)\n5. [Post-Deployment](#post-deployment)\n\n---\n\n## Frontend Deployment (Vercel)\n\n### Prerequisites\n- GitHub repository with code pushed\n- Supabase project created\n- Backend API deployed\n\n### Step 1: Prepare Frontend\n\n1. **Build configuration** - Already configured in `vite.config.js`\n\n2. **Environment variables** - Create `.env.local`:\n   ```bash\n   VITE_SUPABASE_URL=https://your-project.supabase.co\n   VITE_SUPABASE_ANON_KEY=your-anon-key\n   VITE_BACKEND_URL=https://your-backend.railway.app\n   ```\n\n### Step 2: Deploy to Vercel\n\n1. **Connect to Vercel**\n   - Go to vercel.com\n   - Click \"Import Project\"\n   - Select your GitHub repository\n   - Vercel auto-detects Vite configuration\n\n2. **Configure Build Settings**\n   - Framework: Vite\n   - Build Command: `npm run build`\n   - Output Directory: `dist`\n   - Install Command: `npm ci`\n\n3. **Add Environment Variables**\n   In Vercel dashboard (Settings > Environment Variables):\n   ```\n   VITE_SUPABASE_URL\n   VITE_SUPABASE_ANON_KEY\n   VITE_BACKEND_URL\n   ```\n\n4. **Deploy**\n   - Click \"Deploy\"\n   - Wait for build to complete (~3-5 minutes)\n   - Your app is live at `your-project.vercel.app`\n\n### Frontend Deployment Troubleshooting\n\n**Build fails with module not found**\n- Check all imports are correct\n- Ensure dependencies in `package.json`\n- Clear npm cache: `npm cache clean --force`\n\n**Blank page or 404 errors**\n- Check environment variables are set\n- Verify backend URL is accessible\n- Check browser console for errors\n\n**Supabase connection fails**\n- Verify URL and anon key are correct\n- Check Supabase project is running\n- Enable anonymous sign-in in Supabase dashboard\n\n---\n\n## Backend Deployment (Railway)\n\n### Prerequisites\n- Railway account (railway.app)\n- GitHub repository with backend code\n- Supabase PostgreSQL database\n- Redis database (Railway addon)\n- Stripe account (for payments)\n- OpenAI API key (for AI features)\n\n### Step 1: Prepare Backend\n\n1. **Install dependencies**\n   ```bash\n   cd backend\n   npm install\n   ```\n\n2. **Create `.env` file with all variables**\n   ```bash\n   cp .env.example .env\n   # Fill in all values\n   ```\n\n3. **Test locally**\n   ```bash\n   npm run dev\n   # Should start on http://localhost:5000\n   ```\n\n### Step 2: Deploy to Railway\n\n1. **Connect GitHub**\n   - Go to railway.app\n   - Click \"New Project\"\n   - Select \"Deploy from GitHub repo\"\n   - Authorize GitHub\n   - Select your repository\n\n2. **Configure Service**\n   - Railway auto-detects Node.js\n   - Click on service → Settings\n   - Set start command: `npm start`\n\n3. **Add PostgreSQL Database**\n   - Click \"+ Add Service\"\n   - Search and select \"PostgreSQL\"\n   - Railway creates database automatically\n   - Copy connection string\n\n4. **Add Redis Cache**\n   - Click \"+ Add Service\"\n   - Search and select \"Redis\"\n   - Railway creates Redis automatically\n   - Copy Redis URL\n\n5. **Set Environment Variables**\n   - In Railway dashboard, click on Node service\n   - Go to Variables tab\n   - Add all from `.env.example`:\n\n   ```\n   SUPABASE_URL=https://your-project.supabase.co\n   SUPABASE_ANON_KEY=your-anon-key\n   SUPABASE_SERVICE_KEY=your-service-key\n   SUPABASE_JWT_SECRET=your-jwt-secret\n   POSTGRES_URL=postgresql://user:pass@localhost:5432/db\n   REDIS_URL=redis://localhost:6379\n   PORT=5000\n   NODE_ENV=production\n   STRIPE_SECRET_KEY=sk_live_xxxx\n   STRIPE_WEBHOOK_SECRET=whsec_xxxx\n   OPENAI_API_KEY=sk-xxxx\n   OPENAI_ORG_ID=org-xxxx\n   SENDGRID_API_KEY=SG.xxxx\n   FIREBASE_PROJECT_ID=xxxx\n   FIREBASE_PRIVATE_KEY=xxxx\n   GOOGLE_CLIENT_ID=xxxx\n   GOOGLE_CLIENT_SECRET=xxxx\n   LOG_LEVEL=info\n   CORS_ORIGIN=https://your-frontend.vercel.app\n   ```\n\n### Step 3: Run Database Migrations\n\n1. **Get database credentials** from Railway\n   - Click PostgreSQL service\n   - Copy connection string\n\n2. **Connect via psql**\n   ```bash\n   psql postgresql://user:password@host:5432/railway\n   ```\n\n3. **Run migrations** (in order):\n   ```sql\n   -- Copy contents of backend/migrations/001_init_schema.sql\n   -- Paste and execute\n   ```\n\n   ```sql\n   -- Copy contents of backend/migrations/002_rls_policies.sql\n   -- Paste and execute\n   ```\n\n   ```sql\n   -- Copy contents of backend/migrations/003_triggers.sql\n   -- Paste and execute\n   ```\n\n   Alternatively via command line:\n   ```bash\n   psql postgresql://user:pass@host:5432/db < migrations/001_init_schema.sql\n   psql postgresql://user:pass@host:5432/db < migrations/002_rls_policies.sql\n   psql postgresql://user:pass@host:5432/db < migrations/003_triggers.sql\n   ```\n\n4. **Seed initial data** (optional)\n   ```bash\n   npm run seed\n   ```\n\n### Backend Deployment Troubleshooting\n\n**Build fails**\n- Check Node version >= 18\n- Ensure all dependencies in package.json\n- Check for syntax errors: `npm run lint`\n\n**Database connection error**\n- Verify SUPABASE_URL and credentials\n- Check PostgreSQL is accessible\n- Ensure all migrations ran successfully\n\n**Environment variables not loaded**\n- Click service → Variables\n- Verify all critical vars are set\n- Redeploy after adding variables\n\n**Redis connection error**\n- Verify REDIS_URL is correct\n- Check Redis service is running\n- Test with: `redis-cli -u \"$REDIS_URL\" ping`\n\n**API returns 500 errors**\n- Check logs: Railway → Logs tab\n- Look for database or environment errors\n- Verify all external service keys are valid\n\n---\n\n## Database Setup\n\n### Supabase Configuration\n\n1. **Create Supabase Project**\n   - Go to supabase.com\n   - Create new project\n   - Save database password\n   - Wait for database provisioning (~2 minutes)\n\n2. **Enable Required Modules**\n   - Go to SQL Editor\n   - Enable required PostgreSQL extensions:\n   ```sql\n   CREATE EXTENSION IF NOT EXISTS \"uuid-ossp\";\n   CREATE EXTENSION IF NOT EXISTS \"pgcrypto\";\n   CREATE EXTENSION IF NOT EXISTS \"pgjwt\";\n   ```\n\n3. **Row-Level Security**\n   - Supabase auto-enables RLS\n   - All policies defined in migrations\n   - Test via SQL:\n   ```sql\n   SELECT * FROM roles;\n   ```\n\n4. **Storage Buckets**\n   - Create in Supabase dashboard → Storage\n   - Bucket names (create all as public):\n     - `avatars`\n     - `covers`\n     - `songs`\n     - `thumbnails`\n     - `videos`\n     - `artists`\n     - `channels`\n     - `chat`\n     - `voice`\n     - `documents`\n     - `temp`\n\n5. **Enable Realtime**\n   - Supabase dashboard → Realtime\n   - Enable for tables: songs, playlists, users, messages, notifications\n\n### Remote PostgreSQL Connection\n\nConnect from your local machine:\n```bash\npsql -h project.supabase.co -p 5432 -d postgres -U postgres\n```\n\n---\n\n## Environment Configuration\n\n### Frontend (.env.local)\n```bash\n# Supabase\nVITE_SUPABASE_URL=https://your-project.supabase.co\nVITE_SUPABASE_ANON_KEY=your-anon-key\n\n# Backend API\nVITE_BACKEND_URL=https://your-backend.railway.app\n\n# Optional services\nVITE_FCM_PUBLIC_KEY=your-fcm-key\nVITE_GOOGLE_CLIENT_ID=your-google-id\n```\n\n### Backend (.env)\n```bash\n# Database\nSUPABASE_URL=https://your-project.supabase.co\nSUPABASE_ANON_KEY=your-anon-key\nSUPABASE_SERVICE_KEY=your-service-key\nSUPABASE_JWT_SECRET=your-jwt-secret\n\n# Server\nPORT=5000\nNODE_ENV=production\n\n# Services\nSTRIPE_SECRET_KEY=sk_live_xxxx\nOPENAI_API_KEY=sk-xxxx\n\n# CORS\nCORS_ORIGIN=https://your-frontend.vercel.app\n```\n\n---\n\n## Post-Deployment\n\n### Health Checks\n\n1. **Frontend**\n   ```bash\n   curl https://your-frontend.vercel.app\n   # Should return HTML\n   ```\n\n2. **Backend**\n   ```bash\n   curl https://your-backend.railway.app/health\n   # Should return {\"status\":\"ok\"}\n   ```\n\n3. **Database**\n   ```bash\n   curl https://your-backend.railway.app/api/health/db\n   # Should return {\"status\":\"connected\"}\n   ```\n\n### Testing\n\n1. **User Registration**\n   ```bash\n   curl -X POST https://your-backend.railway.app/api/auth/register \\\n     -H \"Content-Type: application/json\" \\\n     -d '{\"email\":\"test@example.com\",\"password\":\"Test123!\",\"username\":\"testuser\"}'\n   ```\n\n2. **Upload Song**\n   - Use Postman collection: `backend/MileyMusics-API.postman_collection.json`\n   - Or use frontend UI at https://your-frontend.vercel.app\n\n### Monitoring\n\n1. **Railway Logs**\n   - Dashboard → Service → Logs\n   - Filter by level (error, warning, info)\n\n2. **Supabase Monitoring**\n   - Database → Query Performance\n   - Check slow queries\n   - Monitor connection usage\n\n3. **Error Tracking**\n   - Backend logs at `/logs/error.log`\n   - Check for failed migrations or API errors\n\n### Performance Optimization\n\n1. **Database**\n   - Enable query cache for frequently accessed data\n   - Monitor connection count\n   - Archive old logs periodically\n\n2. **Frontend**\n   - Enable Vercel analytics\n   - Monitor Core Web Vitals\n   - Check image optimization\n\n3. **Backend**\n   - Monitor Redis memory usage\n   - Check queue processing times\n   - Scale if needed\n\n### Scaling\n\n**If traffic increases:**\n\n1. **Railway**\n   - Click service → Scale\n   - Increase CPU and RAM\n   - Add multiple instances for load balancing\n\n2. **Database**\n   - Supabase auto-scales\n   - Monitor disk usage\n   - Archive old data if needed\n\n3. **Redis**\n   - Railway auto-scales\n   - Monitor hit rate\n   - Consider dedicated Redis instance\n\n---\n\n## Rollback Procedures\n\n### Frontend Rollback (Vercel)\n- Go to Deployments tab\n- Click \"Promote to Production\" on previous version\n- Takes ~2 minutes\n\n### Backend Rollback (Railway)\n- Click service → Deployments\n- Select previous deployment\n- Click \"Redeploy\"\n- Takes ~5 minutes\n\n### Database Rollback\n- Keep migration backups\n- Create snapshot before major changes\n- Use Supabase point-in-time recovery if needed\n\n---\n\n## Support\n\n- **Vercel Docs**: https://vercel.com/docs\n- **Railway Docs**: https://docs.railway.app\n- **Supabase Docs**: https://supabase.com/docs\n- **Express Docs**: https://expressjs.com\n\n---\n\n## Checklist\n\n- [ ] Frontend environment variables set\n- [ ] Backend environment variables set\n- [ ] Database migrations applied\n- [ ] Storage buckets created\n- [ ] Stripe webhook configured\n- [ ] Email service configured\n- [ ] Firebase configured\n- [ ] Google OAuth configured\n- [ ] SSL certificates valid\n- [ ] Health checks passing\n- [ ] Monitoring set up\n- [ ] Backups configured\n- [ ] Domain name configured\n- [ ] Analytics enabled\n- [ ] Error tracking enabled\n
